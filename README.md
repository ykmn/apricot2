# Radio Monitor

Веб-приложение для навигации по записям контроля радиоэфира, поиска нужного момента, выделения фрагментов и экспорта в аудиоформат.

## Требования

- Python 3.9+
- **ffmpeg** — должен быть установлен и доступен в PATH (или путь указан в `config/settings.yaml`)
  - macOS: `brew install ffmpeg`
  - Ubuntu/Debian: `apt install ffmpeg`
  - Windows: https://ffmpeg.org/download.html

## Установка

```bash
pip install -r requirements.txt
```

## Конфигурация

### Структура папок конфигурации

```
config/
├── settings.yaml           # Глобальные настройки (порт, ffmpeg, timezone)
├── stations/
│   ├── retro_fm.yaml       # Одна станция = один файл
│   └── europa_plus.yaml
└── playlists/
    ├── retro_fm_plog.yaml  # Плейлог-источник = один файл
    └── europa_plus_plog.yaml
```

### Аутентификация SMB

Пароли хранятся в переменных окружения (не в YAML-файлах).  
Создайте файл `.env` или экспортируйте переменные:

```bash
export RETRO_FM_PASSWORD="secret"
export EUROPA_PASSWORD="secret"
export PLAYOUT_PASSWORD="secret"
```

Или скопируйте `.env.example` → `.env` и заполните значения, затем:
```bash
source .env   # Linux/Mac
```
На Windows — настройте через «Переменные среды» в системных свойствах.

### Локально смонтированные сетевые папки

Если SMB-шара уже смонтирована (буква диска на Windows, точка монтирования на Linux/Mac),
укажите `local_path` вместо `smb:` в конфиге станции:

```yaml
channels:
  - id: retro_fm_pgm_hq
    name: "Retro FM — PGM-HQ"
    local_path: "/mnt/retro_fm/PGM-HQ"   # Linux/Mac
    # local_path: "Z:\\PGM-HQ"           # Windows
    folder_format: "%Y-%m-%d"
    file_format: "%H-%M-%S"
    file_extension: "wav"
    sample_rate: 44100
```

### Формат имён файлов

Используется синтаксис `strftime`. Примеры:
- `"%Y-%m-%d"` → `2026-06-02`  (папки)
- `"%H-%M-%S"` → `18-24-06`   (файлы)
- `"%d%m%y"`   → `020626`

### Плейлог (CSV)

```yaml
file_mask: "%Y-%m-%d.csv"   # имя файла по дате
delimiter: ";"
fields:
  date:     "Date"          # колонка с датой (или пусто, если дата в имени файла)
  time:     "Time"          # колонка с временем HH:MM:SS
  title:    "Title"         # название элемента
  cls:      "Class"         # класс: M=музыка, J=джингл, A=реклама, P=передача
  duration: "Duration"      # длительность в секундах (необязательно)
```

## Запуск

```bash
python run.py
```

Откройте браузер: **http://localhost:8765**

## Использование

### Навигация по таймлайну

| Ряд | Содержание | Прокрутка |
|-----|-----------|-----------|
| 1 | Секунды | Медленно — по секундам |
| 2 | Минуты | Средне |
| 3 | Часы | Быстро |
| 4 | Дни | Очень быстро — по дням |
| 5 | Месяцы | По месяцам |

- **Перетащите** любой ряд влево/вправо для перемотки
- **Колёсико мыши** на нужном ряду
- **⟳** — прыжок к текущему времени
- Клик по числу в часах/датах/месяцах — прыжок к этому моменту

### Цветовая индикация

- **Синие ячейки** — есть запись в этот момент
- **Тёмные ячейки** — запись отсутствует
- **Жёлтая подсветка** — выделенный сегмент

### Выделение фрагмента

1. Навигируйте к началу нужного фрагмента
2. Нажмите **◀|** (установить начало)
3. Перемотайте к концу фрагмента
4. Нажмите **|▶** (установить конец)
5. Нажмите **Добавить в лог-лист**

### Экспорт

- Нажмите **↓** рядом с записью в лог-листе
- Выберите формат (MP3 / WAV / AAC) и битрейт
- Нажмите «Экспортировать» — файл скачается автоматически
- **Скачать всё** — экспортирует все записи лог-листа последовательно

### Воспроизведение

- **▶** — воспроизведение с текущей позиции (или с начала выделения)
- **⏸** — пауза
- Во время воспроизведения центральная линия таймлайна движется вперёд

## Архитектура

```
radio-monitor/
├── app/
│   ├── main.py          # FastAPI-приложение, REST API, WebSocket
│   ├── config.py        # Загрузка YAML-конфигов
│   ├── smb_client.py    # Единый слой доступа: локальный путь или SMB
│   ├── file_index.py    # Индекс аудиофайлов в памяти, polling
│   ├── audio.py         # Стриминг и экспорт через ffmpeg
│   ├── playlist.py      # Чтение CSV-плейлогов
│   └── models.py        # Dataclass-модели
├── static/
│   ├── index.html
│   ├── style.css
│   ├── timeline.js      # Многоуровневый таймлайн (canvas)
│   └── app.js           # Главная логика UI
└── config/              # YAML-конфиги (редактируются вручную)
```

### WebSocket-обновления

Сервер следит за появлением/удалением файлов (polling каждые N секунд, настраивается в `settings.yaml`). При изменении браузер получает WS-сообщение и мгновенно обновляет цветовую индикацию таймлайна без перезагрузки.

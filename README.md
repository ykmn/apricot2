# <img src="./static/favicon.svg" width="24"/> Авокадо

Веб-приложение для навигации по записям контроля радиоэфира: поиск нужного момента, выделение фрагментов и экспорт в аудиоформат.

## Требования

- Python 3.9+
- [**ffmpeg**](https://ffmpeg.org/download.html) — должен быть установлен и доступен в PATH (или путь указан в `config/settings.yaml`)
  - macOS: `brew install ffmpeg`
  - Ubuntu/Debian: `apt install ffmpeg`
  - Windows: `winget install ffmpeg`

## Установка

```bash
pip install -r requirements.txt
```

## Конфигурация

### Структура папок

```
config/                         # Рабочая конфигурация (создаётся вами, не в репо)
├── settings.yaml               # Глобальные настройки
├── secret.yaml                 # Учётные данные SMB (не коммитить!)
├── stations/
│   ├── retro_fm.yaml           # Одна станция = один файл
│   └── europa_plus.yaml
└── playlogs/
    ├── retro_fm.yaml      # Плейлог-источник = один файл
    └── europa_plus.yaml

config.demo/                    # Демо-конфигурация (в репо, без паролей)
```

Скопируйте `config.demo/` → `config/` как отправную точку:
```bash
cp -r config.demo/ config/
```

---

### Учётные данные SMB (`config/secret.yaml`)

Все логины, пароли и домены хранятся **в одном файле** `config/secret.yaml`, на который ссылаются конфиги станций по числовому `id`. Файл добавлен в `.gitignore` и никогда не попадает в репозиторий.

```yaml
# config/secret.yaml
authorization:
  - id: 1
    username: "logger"
    password: "password1"
    domain: "CORP"
  - id: 2
    username: "svc_radio"
    password: "another_pass"
    domain: "CONTOSO"
```

> Альтернатива для отдельных каналов — `password_env: "MY_ENV_VAR"` (пароль из переменной окружения).

### Конфигурация станций

#### Подключение через SMB (network share)

Используйте `secret: N`, где N — `id` из `config/secret.yaml`:

```yaml
# config/stations/retro_fm.yaml
id: retrofm
name: "Retro FM"

channels:
  - id: retrofm_pgm
    name: "Retro FM — PGM (mp3/64k)"
    smb:
      host: "fileserver.domain.local"   # имя или IP сервера
      share: "LOGGER"                   # имя сетевой шары
      path: "04 Retro - PGM1"           # подпапка внутри шары (необязательно)
      secret: 1                         # ссылка на запись в secret.yaml
    folder_format: "%Y-%m-%d"
    file_format: "%H-%M-%S"
    file_extension: "mp3"
    sample_rate: 44100
    playlists:
      - retrofm
```

Путь на сервере формируется как: `\\host\share\path\<папка-даты>\<файл>`.

Можно также указать учётные данные прямо в конфиге канала (не рекомендуется):
```yaml
smb:
  host: "fileserver"
  share: "LOGGER"
  path: "04 Retro - PGM1"
  username: "logger"
  password: "secret"        # или password_env: "MY_VAR"
  domain: "CORP"
```

#### Локально смонтированная папка

Если SMB-шара уже смонтирована как буква диска (Windows) или точка монтирования (Linux/Mac):

```yaml
channels:
  - id: retro_fm_pgm_hq
    name: "Retro FM — PGM-HQ (wav/48000)"
    local_path: "/mnt/retro_fm/PGM-HQ"   # Linux/Mac
    # local_path: "Z:\\PGM-HQ"           # Windows (без двойных слешей в начале, но с двойными слешами в пути к папке)
    folder_format: "%Y-%m-%d"
    file_format: "%H-%M-%S"
    file_extension: "wav"
    sample_rate: 48000
```

#### Формат имён файлов

Используется синтаксис `strftime`:

| Параметр | Пример значения | Результат |
|---|---|---|
| `folder_format: "%Y-%m-%d"` | папки | `2026-06-03` |
| `file_format: "%H-%M-%S"` | файлы | `18-24-06.mp3` |
| `file_format: "%d%m%y_%H%M"` | файлы | `030626_1824.wav` |

#### Битрейт (важно для MP3)

Если MP3-файлы имеют нестандартный битрейт в сочетании с JointStereo, укажите битрейт явно — иначе длительность файлов будет рассчитана неверно:

```yaml
file_extension: "mp3"
bitrate: "64k"    # 32k, 64k, 128k, 192k, 320k
```

При первом сканировании приложение пытается определить битрейт автоматически через `ffprobe` и сохраняет результат в `config/detected_params.yaml`.

### Плейлог

Каждый файл конфигурации описывает **один плейлог-источник** с одним или несколькими источниками данных и приоритетами между ними. Если на нужную дату нет данных в источнике с `priority: 1`, автоматически используется `priority: 2` и т.д.

```yaml
# config/playlogs/retro_fm.yaml
id: retrofm_plog
name: "Retro FM — Плейлог"

sources:
  - priority: 1
    local_path: "/mnt/retro-air-1/playlogs"   # основная эфирная машина
    file_mask: "%Y-%m-%d.log"
    encoding: "windows-1251"
    delimiter: ","
    header_skip_prefix: "FIELD LIST"           # первый столбец строки-заголовка (пропустить)
  - priority: 2
    smb:                                       # резервная машина через SMB
      host: "retro-air-2.domain.local"
      share: "playlogs"
      secret: 2
    file_mask: "%Y-%m-%d.log"
    encoding: "windows-1251"
    delimiter: ","
    header_skip_prefix: "FIELD LIST"

# Маппинг полей файла → внутренние поля
fields:
  datetime:  "EventTime"      # объединённое поле даты и времени
  title:     "ElemName"       # название трека / элемента
  artist:    "ElemArtist"     # исполнитель (если пусто — только ElemName)
  cls:       "ElemClass"      # класс: M/J/C/P/N
  db_id:     "ElemDbId"       # ID в базе данных → всплывает при наведении
  id_number: "ElemIdNumber"   # порядковый номер → всплывает при наведении

class_colors:
  M: "#1565c0"
  J: "#558b2f"
  C: "#e65100"
  P: "#6a1b9a"
  N: "#37474f"
  default: "#424242"

class_names:
  M: "Музыка"
  J: "Джингл"
  C: "Реклама"
  P: "Передача"
  N: "Новости"
```

**Формат строки плейлога в UI:**  
`Исполнитель — Название` (или просто `Название`, если исполнитель не заполнен).  
При наведении на строку показывается всплывающая подсказка: `[dbID: 70501 // ID_Number: R071-01]`.

**Кэш плейлогов** хранится в `cache_playlogs/{playlist_id}/{YYYY-MM-DD}.json`.  
Данные прошлых дат кэшируются навсегда; сегодняшний день не кэшируется (файл может дописываться).

### Глобальные настройки (`config/settings.yaml`)

```yaml
server:
  host: "0.0.0.0"
  port: 8765

watcher:
  poll_interval: 10        # интервал опроса SMB-шар (секунд)

export:
  default_format: "mp3"
  default_bitrate: "192k"
  retention_days: 3        # экспортированные файлы удаляются через N дней
  cleanup_time: "03:00"    # время ежедневной очистки папки export/ (HH:MM)

ffmpeg_path: "ffmpeg"      # путь к ffmpeg (или просто "ffmpeg" если в PATH)

ui:
  timezone: "Europe/Moscow"
```

Папка `export/` очищается автоматически: при старте приложения и ежедневно в `cleanup_time`.

## Использование

### Выбор канала

В верхней панели:

- Поле **Фильтр…** — введите часть названия для быстрого поиска
- Кнопка **▾** — открыть полный список каналов (сгруппирован по станциям)
- Кнопка **📋** — скопировать путь к папке канала в буфер обмена  
  (`\\server\share\path` для SMB, локальный путь для смонтированных шар)

### Навигация по таймлайну

| Ряд | Масштаб | Прокрутка |
|-----|---------|-----------|
| 1 | Секунды | Посекундно |
| 2 | Минуты | Поминутно |
| 3 | Часы | Почасово |
| 4 | Дни | Подневно |
| 5 | Месяцы | Помесячно |

- **Перетащите** любой ряд влево/вправо для перемотки
- **Колёсико мыши** — прокрутка на нужном ряду
- **⟳** — прыжок к текущему времени
- Горячие клавиши: **`S`** — фокус на поле выбора канала; **`[`** / **`]`** — метка In/Out; **`0`** — Play/Pause; **`+`** / **`-`** — шаг по строке таймлайна

### Цветовая индикация

- **Синие ячейки** — запись присутствует
- **Тёмные ячейки** — запись отсутствует
- **Жёлтая подсветка** — выделенный сегмент

### Выделение фрагмента

1. Навигируйтесь к началу нужного момента
2. Нажмите **In** (установить начало)
3. Перемотайте к концу фрагмента
4. Нажмите **Out** (установить конец)
5. Нажмите **Добавить сегмент**

### Экспорт

- Нажмите **↓** рядом с записью в списке сегментов
- В диалоге экспорта:
  - По умолчанию выбирается **оригинальный формат канала** — экспорт выполняется без перекодирования (`-c copy`), максимально быстро и без потерь качества
  - При выборе другого формата — выполняется перекодирование с указанными параметрами
- **Скачать всё** — экспортирует все сегменты из списка последовательно

Имя экспортируемого файла: `{Название канала} YYYY-mm-dd HH-MM-SS.fmt`  
Пример: `Europa Plus PGM 2026-06-03 14-30-00.mp3`

Недопустимые символы в названии канала заменяются пробелом.  
Экспортированные файлы сохраняются в папку `export/` и автоматически удаляются через `retention_days` дней.

### Список выбранных сегментов

Сегменты сохраняются в `localStorage` и **не сбрасываются при обновлении страницы**. При следующем открытии браузера все ранее добавленные сегменты восстанавливаются автоматически.

### Воспроизведение

- **▶** — воспроизведение с текущей позиции (или с начала выделенного фрагмента)
- **⏸** — пауза
- Во время воспроизведения центральная линия таймлайна движется в реальном времени
- **Перемотка во время воспроизведения** — прокрутите таймлайн колёсиком/перетащите или измените дату/время: анимация замирает на новой позиции, через 250 мс начинается буферизация и аудио перескакивает на выбранный момент
- Во время буферизации в строке статуса появляется мигающий индикатор **⌛ Буферизация…**, который гаснет в момент реального начала звука

### Плейлог

Левая панель показывает плейлог для выбранного канала в окне ±3 часа от текущей позиции:

- Каждая строка окрашена цветом класса элемента (настраивается в конфиге плейлога)
- **Текущий элемент** (соответствующий позиции таймлайна) выделен **жирным** и автоматически прокручивается на 1/3 высоты панели
- При наведении на строку всплывает подсказка с ID из базы данных: `[dbID: ... // ID_Number: ...]`
- Кнопка **↑** — добавить элемент плейлога в список выбранных сегментов

### Строка статуса индекса

Под транспортными кнопками — строка аудио-индекса с цветными точками (по одной на канал):

- 🟢 **Зелёная** — канал просканирован, файлы найдены. Подсказка: количество файлов
- 🔴 **Красная** — канал недоступен. **Нажмите на точку** — появится всплывающее окно с причиной ошибки
- ⟳ **Синяя мигающая** — канал сканируется прямо сейчас

Клик по тексту «⚠ N недоступн.» показывает список всех недоступных каналов с ошибками.

### Строка статуса плейлогов

Ниже строки аудио-индекса — строка доступности источников плейлогов (по одной точке на источник):

- 🟢 **Зелёная** — источник доступен (файл за вчерашний день найден)
- 🔴 **Красная** — источник недоступен. **Нажмите на точку** — текст ошибки
- ⟳ **Синяя мигающая** — проверка выполняется

Проверка запускается автоматически после старта и при «Пересканировать плейлоги станции». Информационные сообщения («Плейлоги доступны…») исчезают через 5 секунд; точки остаются.

### Меню (☰)

| Пункт | Действие |
|---|---|
| ☀️ Светлый / 🌙 Тёмный режим | Переключение темы |
| 🔍 Пересканировать текущий канал | Принудительно перечитать список аудиофайлов |
| 📋 Пересканировать плейлоги станции | Очистить кэш плейлогов за последние 2 дня и перепроверить все источники текущей станции |
| 🔄 Обновить конфигурацию | Перечитать YAML-файлы без перезапуска сервера |
| 🔁 Перезапустить сервер | Полный перезапуск процесса; страница обновится автоматически |

## Архитектура

```
radio-monitor/
├── app/
│   ├── main.py          # FastAPI-приложение, REST API, WebSocket
│   ├── config.py        # Загрузка YAML-конфигов, разрешение secret.yaml
│   ├── smb_client.py    # Единый слой доступа: локальный путь или SMB
│   ├── file_index.py    # Индекс аудиофайлов в памяти, polling, кэш
│   ├── audio.py         # Стриминг и экспорт через ffmpeg (copy / transcode)
│   ├── audio_probe.py   # Автоопределение параметров через ffprobe
│   ├── cache.py         # Дисковый кэш аудио-индекса (cache_audio/*.json)
│   ├── playlist.py      # Чтение CSV-плейлогов
│   └── models.py        # Dataclass-модели
├── static/
│   ├── index.html
│   ├── style.css
│   ├── timeline.js      # Многоуровневый таймлайн (canvas)
│   └── app.js           # Главная логика UI
├── config/              # Рабочая конфигурация (не в репо)
├── config.demo/         # Демо-конфигурация (в репо)
├── export/              # Экспортированные файлы (auto-cleanup)
├── cache_audio/         # Дисковый кэш индекса аудиофайлов (auto-generated)
└── cache_playlogs/      # Дисковый кэш плейлогов по датам (auto-generated)
```

### Индекс файлов и кэш

При запуске приложение:
1. Загружает индекс из `cache/*.json` (мгновенно) — таймлайн сразу показывает данные
2. В фоне сканирует SMB-шары и обновляет индекс — `scandir()` читает размер файлов из ответа SMB-сервера без отдельных `stat`-запросов на каждый файл
3. Сохраняет обновлённый индекс обратно в кэш

### WebSocket-обновления

Сервер следит за появлением/удалением файлов (polling каждые N секунд, настраивается через `watcher.poll_interval`). При изменении браузер получает WS-сообщение и мгновенно обновляет цветовую индикацию без перезагрузки страницы.

## 0. Создание приложения с нуля

*Первая сессия, Mac. Проект тогда назывался «radio-monitor», v0.1.001.*

> Мне нужно веб-приложение для поиска нужного времени в записях контроля радиоэфира, выделения и экспорта фрагмента. Радиостанций и каналов может быть несколько, настройки в YAML. На экране: области прокрутки (секунды, минуты, часы, дни, месяцы), выбор канала с поиском, плейлог, список сегментов.

**Создана вся базовая архитектура:**

| Модуль | Функция |
|--------|---------|
| `smb_client.py` | Единый API для локальных путей и SMB (`smbprotocol`) |
| `file_index.py` | In-memory индекс файлов, polling, WS-уведомления |
| `audio.py` | Стриминг и экспорт через ffmpeg (конкатенация, обрезка) |
| `playlist.py` | Чтение CSV с гибким маппингом полей |
| `main.py` | REST API + WebSocket |

**`static/timeline.js`** — canvas-таймлайн, 5 рядов (секунды → месяцы), синхронизированы через `centerTime`. Синяя подсветка = есть аудио, жёлтая = выделенный сегмент.

**Первый коммит** — инициализирован git-репозиторий, перенесён на GitHub, продолжение разработки на Windows.

### SMB-монтирование на Linux, диагностика

> Сетевая папка не монтируется на Ubuntu 26.04.

```python
sec = "krb5" if smb.auth_protocol == "kerberos" else "ntlmssp"
base_options = [..., f"sec={sec}"]
```

```python
dl_tasks = [asyncio.create_task(_stage_one(i, af)) for i, af in enumerate(files)]
for i, (af, dl_task) in enumerate(zip(files, dl_tasks)):
    local_path = await dl_task   # ждём только этот файл
    # ffmpeg стартует немедленно
```

**1. Несовместимость naive/aware datetime — `app/cache.py`**

Предыдущее изменение добавило `tz=timezone.utc` при загрузке дат из дискового кэша (`datetime.fromtimestamp(ts, tz=timezone.utc)`). Это породило несовместимость: `_scan_date()` в `file_index.py` создаёт naive-datetime из имён файлов, route handlers в `main.py` передают `datetime.fromtimestamp(start)` без tz. При совместном нахождении aware и naive объектов в одном `SortedList` — `TypeError` при любом аудио-запросе после загрузки кэша.

Исправление: откат `cache.py` к naive-datetime. Вся система единообразно использует локальное время.

**2. Неверный escape одиночных кавычек в ffconcat — `app/audio.py`**

В `stream_audio` был применён SQL-стиль удвоения кавычек (`'` → `''`), который не является валидным escape для ffconcat: демультиплексор воспринимает первую `'` как конец имени файла. Правильный escape — `'` → `\'`.

В `export_audio` escape отсутствовал вообще — пути с апострофом (например, `Today's Show`) ломали concat-лист.

Исправление: `.replace("'", "\\'")` в обоих местах.

### Прочие исправления

**3. Избыточный `os.chmod` после `mkstemp` — `app/smb_mount.py`**

`tempfile.mkstemp` всегда создаёт файл с правами 0600 вне зависимости от `umask` процесса. Явный `os.chmod(creds_path, 0o600)` был лишним и удалён.

**4. Двойной `import os` внутри функции — `app/auth.py`**

`os` не был импортирован на уровне модуля, поэтому появился дважды внутри `_save_sessions()` — один раз в `try`-блоке, второй в `except`. Вынесен в заголовок модуля.

### Хэширование паролей локальных пользователей

Добавлена поддержка хранения паролей в виде Argon2id-хэшей в `config/users.yaml`:

```yaml
users:
  - username: admin
    password_argon2: "$argon2id$v=19$m=65536,t=3,p=4$..."
    is_admin: true
```

Логика в `auth.py` — функция `_check_local_password()`:
1. Если есть `password_argon2` — проверяет через `argon2-cffi` (`PasswordHasher.verify`)
2. Иначе — сравнение открытого текста через `hmac.compare_digest` (обратная совместимость)

Утилита `tools/hash_password.py` запрашивает пароль интерактивно (скрытый ввод, двойное подтверждение) и выводит готовый YAML-сниппет.

В `requirements.txt` добавлен `argon2-cffi>=23.1.0`.

### Защита secret.yaml правами файловой системы

Пароли SMB/LDAP в `secret.yaml` хранятся в открытом виде — они нужны приложению для подключения к удалённым серверам, и хэш здесь неприменим. Рекомендованная защита — ограничение прав доступа к файлу:

```bash
chmod 600 config/secret.yaml
```

Задокументировано в README.md.

### Не подтверждённые кандидаты

- **timeline.js center line** — `ctx.strokeStyle = TC.centerLine` перемещён после `beginPath()`, но до `stroke()`. Canvas 2D читает стиль в момент `stroke()`, регрессии нет.
- **`asyncio.get_running_loop()`** — все три замены в `file_index.py` находятся внутри `async def`, RuntimeError невозможен.
- **Кэш `auth_required()`** — `invalidate_auth_cache()` корректно вызывается из `_load_config()` при reload.

### Новые API-эндпоинты

Оба эндпоинта защищены `_require_admin(request)`.

**`GET /api/check_updates`**

1. Читает `update.repo_url` из `settings`.
2. Извлекает `owner/repo` из URL через регулярное выражение.
3. Запрашивает локальный HEAD через `git log -1`.
4. Запрашивает последний коммит через GitHub API (`/repos/{owner_repo}/commits/HEAD`) с помощью `urllib.request` (stdlib, без новых зависимостей).

1. Выполняет `git pull` через `subprocess.run` с таймаутом 60 с.
2. При ненулевом коде возврата возвращает HTTP 500 с выводом git.
3. Планирует `os._exit(0)` через 0.5 с (аналогично `/api/restart`) — systemd перезапустит процесс.

## 9. Замена argon2 на PBKDF2-SHA256 (stdlib)

*Сессия 2026-06-12, v1.2.044.*

### Проблема

На Python 3.14 `cffi 2.0.0` не может загрузить C-расширение `argon2-cffi-bindings`: при `from argon2 import PasswordHasher` возникает не `ImportError`, а ошибка инициализации `.so`. Результат — вход с Argon2-хэшем невозможен на продакшн-сервере.

### Решение

Отказ от внешней зависимости `argon2-cffi` в пользу `hashlib.pbkdf2_hmac` (Python stdlib, работает на LibreSSL и OpenSSL, CPython любой версии).

**Формат хэша** — `$pbkdf2$sha256$<iterations>$<salt_base64>$<hash_base64>`  
**Параметры** — PBKDF2-SHA256, 600 000 итераций (рекомендация OWASP 2023 для PBKDF2-SHA256), соль 32 байта, dklen 32 байта.

### Изменения

| Файл | Что изменилось |
|---|---|
| `app/auth.py` | `_hash_pbkdf2()`, `_verify_pbkdf2()`; новое поле `password_pbkdf2` с приоритетом 1 |
| `app/auth.py` | Argon2-блок сохранён как legacy (приоритет 2), с корректным fallback при ошибке импорта |
| `tools/hash_password.py` | Переписан на PBKDF2, больше не требует argon2-cffi |
| `requirements.txt` | Строка `argon2-cffi>=23.1.0` удалена |
| `config/users.yaml` | Поля `password_argon2` заменены на `password_pbkdf2` |

### Приоритет полей в users.yaml

1. `password_pbkdf2` — PBKDF2-SHA256 (stdlib, рекомендуется)
2. `password_argon2` — Argon2id (legacy, argon2-cffi)
3. `password` — открытый текст (только обратная совместимость)

**Реализация в `apricot2.py`:**

- Замена блокирующего `uvicorn.run()` на `asyncio.run(_run_servers())`

- Redirect-приложение — минимальный ASGI-обработчик: читает заголовок `Host`, сохраняет path и query string

**Конфигурация в `settings.yaml`:**

```yaml
server:
  ssl:
    enabled: true
    cert: "ssl/cert.pem"
    key:  "ssl/key.pem"
    http_redirect_port: 80   # 0 или отсутствие ключа — редирект отключён
```

## 11. Фиксы авторизации LDAP и SMB-монтирования

*Сессия 2026-06-16, v1.2.054–v1.2.055.*

### Переподключение при таймауте transitive group search (v1.2.054)

При авторизации через AD возникал таймаут на `LDAP_MATCHING_RULE_IN_CHAIN` запросе. Соединение «умирало», после чего падал и запрос `primaryGroupID` — group search давал WARNING и primary group терялась.

**Изменения в `app/auth.py`:**

- Добавлена `_reconnect(conn)` — переоткрывает и перебиндивает существующее соединение
- `_get_transitive_groups` при первой ошибке переподключается и повторяет запрос; только при второй — возвращает `None` (WARNING)
- При fallback на `direct memberOf` вызывается `_reconnect()` перед `_get_primary_group_dn`, чтобы primary group не терялась из-за мёртвого сокета

### Фиксы SMB-монтирования (v1.2.054)

**`app/smb_mount.py`:**

- `_CIFS_BIN` — путь к `mount.cifs` резолвится один раз при импорте, а не повторно в каждой функции. Исключает расхождение между командой `sudo` и подсказкой sudoers (актуально когда бинарник в `/usr/sbin/` вместо `/sbin/`)

### Устойчивость при сбое HTTP-редиректа (v1.2.055)

**Проблема:** если `authbind` не настроен для порта 80, redirect-сервер бросает `SystemExit(1)`, `asyncio.gather` без изоляции роняет вместе с ним и основной HTTPS-сервер — приложение перезапускалось каждые несколько секунд.

**`apricot2.py`:**

- Redirect-сервер запускается через `_serve_redirect()` — обёртку, которая перехватывает `SystemExit` и `Exception`, логирует в stderr и возвращает управление
- Основной сервер работает независимо: сбой редиректа больше не роняет всё приложение

### SMB pipe-стриминг: мгновенный старт воспроизведения (v1.2.057)

**Проблема:** при воспроизведении с SMB-шары браузер получал первый аудиочанк только после того, как весь файл скачивался целиком во временную папку (для часового эфира — 200–300 МБ, задержка десятки секунд).

**`app/audio.py`:**

- Для чистых SMB-каналов (без локального монтирования) `stream_audio` теперь использует **pipe-режим**: файл открывается через `smb.open_file()`, хэндл сразу seekируется до нужного байтового оффсета, данные читаются чанками (64 КБ) и передаются в **stdin** ffmpeg через `asyncio` pipe
- `_byte_offset_for_seek(ss, channel)` — вычисляет байтовый оффсет для старта: CBR MP3/AAC = `bitrate_bps/8 * ss`, WAV = `44 + sample_rate*2ch*2bytes*ss`. Незапрошенный префикс файла не передаётся по сети совсем
- `_pipe_smb_segment(af, channel, ss, duration, ...)` — корутина-генератор: запускает ffmpeg с `-f <fmt> -i pipe:0 [-t duration]`, фоновой задачей (`feed_stdin`) читает SMB и льёт в stdin; основная сопрограмма читает stdout и yielдит чанки
- При неизвестном битрейте (`byte_offset=0`, `ss>0`) fallback: ffmpeg получает `-ss` и проматывает сам — корректно, но медленнее
- Staging-путь (`_stage_one` + temp dir) сохранён без изменений для локальных и смонтированных каналов
- `export_audio` не затронут — по-прежнему использует `_stage_files` (нужен реальный файл для корректного WAV RIFF-заголовка)

### Персистентное UI-состояние на сервере (v1.2.058)

**Задача:** при закрытии и повторном открытии браузера восстанавливать выбранный канал, позицию таймлайна, маркеры выделения и список фрагментов. Ранее список фрагментов хранился в `localStorage` (теряется в инкогнито, не синхронизируется между устройствами), остальное не сохранялось.

**Новый модуль `app/ui_state.py`:**

- Чтение/запись `config/sessions-ui.yaml` ключом `username` (или `__anonymous__` при отключённой авторизации)
- Атомарная запись через `tempfile.mkstemp` + `os.replace`
- Хранит: `channel_id`, `timeline_time`, `sel_start`, `sel_end`, `log_items`

**`app/main.py`:**

- Хелпер `_current_username(request)` — извлекает имя пользователя из cookie сессии
- `GET /api/ui-state` — возвращает сохранённое состояние для текущего пользователя
- `PUT /api/ui-state` — сохраняет состояние; валидирует и фильтрует допустимые поля

**`static/app.js`:**

- `_fetchUiState()` — загружает состояние с сервера при старте страницы
- `_scheduleUiStateSave()` / `_doSaveUiState()` — дебаунс 1500 мс; вызывается при смене канала, изменении выделения, изменении списка фрагментов
- `visibilitychange` → мгновенное сохранение при сворачивании/закрытии вкладки (чтобы позиция таймлайна не терялась)
- URL-параметры `?ch=` и `?t=` имеют приоритет над сохранённым состоянием
- **Миграция:** при первом открытии после обновления данные из `localStorage` (`apricot-logitems`) переносятся на сервер, запись в `localStorage` удаляется
- `_saveLogItems()` и `_loadLogItems()` удалены; вместо них — единый серверный механизм

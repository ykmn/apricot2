# Мобильный интерфейс — разработка (2026-06-11)

Версия: 1.2.039 → **1.2.040**  
Коммит: `f12fbe2`

---

## Задача

Адаптировать веб-интерфейс для вертикального экрана смартфона.  
Требования: уменьшить таймлайн, плейлог 6 видимых строк, список фрагментов ниже плейлога, вертикальная прокрутка.

---

## Этап 1 — Три варианта макета (прототипирование)

Созданы HTML-макеты (`static/mobile-mockups.html`, удалён перед коммитом) с тремя концепциями:

| Вариант | Концепция | Особенность |
|---------|-----------|-------------|
| **А** | Стандартная прокрутка | 5 рядов таймлайна × 30 px, плейлог 15 строк, всё прокручивается |
| **Б** | Ультракомпакт | Топбар + канал в одну строку, 5 рядов × 24 px, транспорт в одну строку |
| **В** | Sticky-таймлайн + вкладки | Таймлайн и транспорт зафиксированы, прокручивается только список |

**Выбран Вариант В.**

---

## Этап 2 — Уточнение структуры топбара

По запросу пользователя топбар разбит на 4 отдельных строки:

1. Логотип + «Абрикос 2» + кнопка ☰
2. Название канала (крупно)
3. Поле фильтра + кнопки ▾ ⧉ #
4. `<input type="date">` + `<input type="time" step="1">` + кнопка ⟳

Поля date/time — нативные HTML-элементы, на мобильном вызывают системный picker с барабанной прокруткой.

Подписи «ДАТА» / «ВРЕМЯ» убраны как лишние.

## Этап 3 — Реализация

### `static/index.html`

- Добавлен `#mob-clock-bar` (дочерний `#channel-bar`, **не** внутри `#channel-controls`)
- Добавлен `#panel-tabs` с кнопками «ПЛЕЙЛОГ» / «ФРАГМЕНТЫ» и `#tab-badge`
- Оба элемента без класса `hidden` — скрываются через CSS `display: none` по умолчанию

> **Важно:** попытка использовать `class="hidden"` ломалась из-за `!important` в `.hidden { display: none !important }` — медиа-запрос не мог его перебить.

### `static/style.css`

Добавлены стили:
- `#mob-clock-bar`, `#mob-date`, `#mob-time`, `#mob-btn-now` — поля ввода, стилизованные под кнопки-спаны часов
- `#panel-tabs`, `.panel-tab`, `.tab-badge` — вкладки с подчёркиванием активной
- `@media (max-width: 640px)` — весь мобильный лейаут:
  - топбар 52 px, логотип 28 px, название 22 px
  - 3 строки channel-bar (название / фильтр+кнопки / дата-время)
  - таймлайн `.tl-row { height: 30px }`
  - транспорт `flex-wrap: wrap` → две строки
  - `#panel-tabs { display: flex }`, `.panel-tab { font-size: 11px }` (без переноса с бейджем)
  - `#playlist-panel .scrollable-list { max-height: calc(6 * 29px) }` — 6 строк видно
  - `#loglist-panel .scrollable-list { max-height: calc(6 * 45px) }`

### `static/app.js`

Новые функции:

```js
_syncMobClock(ts)    // обновляет #mob-date и #mob-time при движении таймлайна
initMobClock()       // слушает change на инпутах → Timeline.setTime()
initMobileTabs()     // переключение вкладок + скрытие неактивной панели
_updateTabBadge()    // обновляет счётчик на вкладке «Фрагменты»
```

Вызовы:
- `_syncMobClock(ts)` добавлен в `onTimeChange` callback
- `initMobClock()`, `initMobileTabs()` вызываются из `DOMContentLoaded`
- `_updateTabBadge()` вызывается из `renderLogList()`

## Этап 4 — Багфикс: подсказка в пустом списке фрагментов

**Симптом:** при первом запуске (пустой localStorage) список «Выбранные фрагменты» пуст без подсказки; подсказка появлялась только после нажатия «Очистить».

**Причина:** `_loadLogItems()` вызывал `renderLogList()` только внутри блока `if (raw)`.

```js
// БЫЛО
function _loadLogItems() {
  try {
    const raw = localStorage.getItem(_LOG_KEY);
    if (raw) { logItems = JSON.parse(raw); renderLogList(); }
  } catch(e) {}
}

// СТАЛО
function _loadLogItems() {
  try {
    const raw = localStorage.getItem(_LOG_KEY);
    if (raw) logItems = JSON.parse(raw);
  } catch(e) {}
  renderLogList(); // всегда, даже при пустом localStorage
}
```

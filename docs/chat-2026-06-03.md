> Дата-время версии должны отражать время последнего внесения изменений, а не последнего коммита

Заменил `git log -1 --format=%ci` на обход `git ls-files` с получением максимального `st_mtime` среди всех файлов под контролем версий:

```python
def _build_date() -> str:
    """Return mtime of the most recently modified tracked file in the project."""
    try:
        r = subprocess.run(["git", "ls-files"], ...)
        if r.returncode == 0 and r.stdout.strip():
            latest = max(
                (PROJECT_ROOT / f).stat().st_mtime
                for f in r.stdout.splitlines()
                if (PROJECT_ROOT / f).exists()
            )
            return datetime.fromtimestamp(latest).strftime("%Y-%m-%d %H:%M")
    except Exception:
        pass
    return datetime.now().strftime("%Y-%m-%d %H:%M")
```

Это точно отражает момент последнего сохранения файла на диске — независимо от того, был ли сделан коммит.

> Давай проведем диагностику почему недоступны файлы канала Europa Plus — PGM (mp3/64k), почему таймаут?

**Шаг 1 — DNS:** оба варианта (`emg-logger2` и `EMG-LOGGER2`) резолвятся в `172.16.230.151`. Одинаково.

**Шаг 2 — TCP 445:** `TcpTestSucceeded: True`. Порт открыт.

**Шаг 3 — прямой тест SMB:**
```
--- Testing: emg-logger2 ---  register_session OK in 0.1s, listdir OK: 120 entries
--- Testing: EMG-LOGGER2 ---  register_session OK in 0.0s, listdir OK: 120 entries
```

**Шаг 4 — полный скан 90 дней:**
```
Done in 1.54s: 10868 files, 0 missing folders out of 91 dates
```

**Нашёл причину:** `_scan_date` вызывал `smb.getsize()` для **каждого** из 10 868 файлов — 10 868 отдельных SMB `stat` запросов.

| Метод | 10 868 файлов |
|-------|--------------|
| `listdir` + `getsize` (было) | ~30–35 с → таймаут |
| `scandir` + `stat()` | 65 с |
| `scandir` + `stat(follow_symlinks=False)` | 45 с |
| `scandir` + `_dir_info.end_of_file` ✓ | **1.7 с** |

`_dir_info.end_of_file` — размер файла уже встроен в ответ SMB `FIND`, без дополнительных round-trip'ов.

**Исправление в `smb_client.py`:**
```python
def scandir(local_path, smb_cfg, rel="") -> list[tuple[str, int]]:
    ...
    for e in smbclient.scandir(unc):
        try:
            size = e._dir_info.end_of_file  # из SMB FIND response, без stat
        except AttributeError:
            size = e.stat(follow_symlinks=False).st_size
        result.append((e.name, size))
```

`file_index.py` — `_scan_date` теперь использует `smb.scandir()` вместо `smb.listdir()` + `smb.getsize()`.

> Снова есть пустота в таймлайне при наличии аудиофайлов. Проверь канал Europa Plus — PGM (mp3/64k), время 3 июня 2026 13:37:54 — после этого времени есть звук, но на таймлайне не отображается

API вернул интервалы: `13:36:00 — 13:37:55` ... `13:48:00 — 14:00:00` — пропуск 10 минут.

На SMB: файл `13-36-00.mp3` сейчас занимает **5 759 232 байт** (12 мин), но в индексе записан с размером ~920 KB (115 сек) — он был захвачен в момент записи.

**Корень проблемы:** `refresh()` сравнивал только пути (`new_paths - self._paths`). Файл, уже существующий в индексе, не обновлялся, даже если его размер вырос.

**Исправление в `file_index.py`:**
```python
# Detect files whose duration changed (e.g. still being recorded when first scanned)
changed_keys: set[str] = set()
existing = {af.rel_path: af for af in self._files}
for key in new_paths & self._paths:
    old_af = existing.get(key)
    if old_af and current[key].end_dt != old_af.end_dt:
        changed_keys.add(key)

added   = [current[k] for k in added_keys | changed_keys]
removed = [af for af in self._files
           if af.rel_path in removed_keys or af.rel_path in changed_keys]
```

Также: ручной ресскан (`/api/rescan`) теперь всегда сбрасывает индекс канала (`idx.clear()`), гарантируя свежие размеры файлов.

> Экспорт сегмента по умолчанию должен предлагаться:
> - для mp3 и aac в оригинальный битрейт канала. Рассмотри ffmpeg-опцию -c (copy без перекодирования)
> - для wav с оригинальной частотой дискретизации канала тоже без перекодирования.
> Перекодирование нужно только при выборе формата отличающегося от оригинального формата канала.

**Логика выбора режима:**

| Формат выбран | Нативный формат канала | Режим |
|---|---|---|
| mp3 | mp3 | `-c copy` — без перекодирования |
| wav | wav | `-c copy` — без перекодирования |
| aac | aac | `-c copy` — без перекодирования |
| wav | mp3 | перекодирование через `pcm_s16le` |
| mp3 | wav | перекодирование через `libmp3lame` |

**`audio.py`** — добавлен параметр `copy_mode`:
```python
if copy_mode:
    fmt = _native_format(channel)   # 'mp3' / 'wav' / 'adts'
    cmd += ["-c", "copy", "-f", fmt]
else:
    if out_format == "wav":
        cmd += ["-acodec", "pcm_s16le", "-f", "wav"]
    elif out_format == "aac":
        cmd += ["-acodec", "aac", "-b:a", bitrate, "-f", "adts"]
    else:
        cmd += ["-acodec", "libmp3lame", "-b:a", bitrate, "-f", "mp3"]
```

**`/api/stations`** теперь возвращает `sample_rate` и `bitrate` канала.

**`app.js`** — при открытии модала:
- Автовыбор нативного формата канала
- Подстановка `sample_rate` и `bitrate` из канала
- Поля частоты и битрейта **задизейблены** в copy-режиме
- Зелёная подсказка «⚡ Оригинальный формат канала — экспорт без перекодирования»
- При смене формата — поля активируются, copy-режим выключается
- `_buildExportBody` передаёт `copy_mode: true` в тело запроса

Проверено: модал для DSP0 (mp3) открывается с MP3, 48 000 Гц, зелёной подсказкой. При переключении на WAV — поля активируются, `copy_mode: false`.
